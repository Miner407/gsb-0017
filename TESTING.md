## 前置准备

1. 确保已安装项目依赖：`npm install`
2. 启动开发服务器：`npm run dev`
3. 在浏览器中打开应用（默认地址：http://localhost:5173）
4. 清空 localStorage 数据（可选，用于干净测试）：
   - 打开浏览器开发者工具 (F12)
   - 切换到 Application/Storage 标签
   - 找到 Local Storage
   - 右键点击站点域名，选择 Clear

## 功能验证清单

### 1. 新增电影

**测试步骤：**
1. 点击「添加新电影」展开表单
2. 在「电影名称」输入框中输入：`盗梦空间`
3. 选择状态为「想看」
4. 评分设为 9.0
5. 选择观影平台：`Netflix`
6. 在短评中输入：`诺兰经典神作`
7. 点击「保存电影」按钮

**预期结果：**
- 表单成功提交并收起
- 电影列表中出现「盗梦空间」卡片
- 状态显示为「想看」
- 统计区「总收藏」数字 +1
- 统计区「想看」数量 +1

### 2. 编辑电影

**测试步骤：**
1. 找到刚才添加的「盗梦空间」电影卡片
2. 点击编辑（铅笔图标）按钮
3. 修改电影名称为：`盗梦空间 (Inception)`
4. 将状态改为「已看」
5. 将评分调整为 9.5
6. 选择观影日期为今天
7. 修改短评为：`诺兰经典神作，层层嵌套的梦境设计`
8. 点击保存（勾选图标）按钮

**预期结果：**
- 编辑模式退出，回到展示模式
- 电影标题更新为新名称
- 状态变为「已看」
- 评分显示为 9.5
- 显示观影日期
- 短评更新为新内容
- 统计区「想看」数量 -1，「已看」数量 +1

### 3. 删除电影

**测试步骤：**
1. 点击某部电影的删除（垃圾桶图标）按钮
2. 观察是否出现确认对话框
3. 点击「取消」按钮
4. 再次点击删除按钮
5. 点击「确认删除」按钮

**预期结果：**
- 第一次点击删除后出现确认对话框
- 点击取消后对话框消失，电影仍然存在
- 第二次确认删除后，电影从列表中消失
- 统计区对应状态的数量 -1
- 总收藏数 -1

### 4. 状态切换

**测试步骤：**
1. 添加一部状态为「想看」的电影
2. 点击电影卡片上的状态按钮（显示「想看 → 已看」）
3. 观察状态变化
4. 再次点击状态按钮（显示「已看 → 弃看」）
5. 再次点击状态按钮（显示「弃看 → 想看」）

**预期结果：**
- 状态按「想看 → 已看 → 弃看 → 想看」循环切换
- 每次切换后，卡片的状态颜色和标签同步更新
- 统计区各状态数量同步更新
- 如果筛选了特定状态，状态切换后电影可能从列表中消失（因为不再匹配筛选条件）
- 当状态变为「已看」且没有观影日期时，建议提示填写日期

### 5. 按状态筛选

**测试步骤：**
1. 确保至少有 3 部电影，分别处于「想看」「已看」「弃看」三种状态
2. 点击筛选区的「想看」按钮
3. 观察电影列表
4. 点击「已看」按钮
5. 观察电影列表
6. 点击「弃看」按钮
7. 观察电影列表
8. 点击「全部」按钮
9. 观察电影列表

**预期结果：**
- 点击「想看」后，列表只显示状态为「想看」的电影
- 点击「已看」后，列表只显示状态为「已看」的电影
- 点击「弃看」后，列表只显示状态为「弃看」的电影
- 点击「全部」后，列表显示所有电影
- 列表上方的计数显示当前筛选结果的数量
- 筛选状态激活时，按钮有高亮样式

### 6. 按评分筛选

**测试步骤：**
1. 确保有多部电影，评分各不相同（例如 6、7.5、8、9、9.5）
2. 在「最低评分」下拉框中选择「8 分以上」
3. 观察电影列表
4. 在「最高评分」下拉框中选择「8 分以下」
5. 观察电影列表
6. 将最低评分设为「不限」，最高评分设为「不限」

**预期结果：**
- 最低评分设为 8 分以上时，只显示评分 ≥ 8 的电影
- 最高评分设为 8 分以下时，只显示评分 ≤ 8 的电影（无评分的电影不显示）
- 重置后显示所有电影
- 列表上方显示「已筛选」提示

### 7. 按平台筛选

**测试步骤：**
1. 确保有多部电影，来自不同平台（Netflix、B站、影院等）
2. 在「平台」下拉框中选择某个平台（如 Netflix）
3. 观察电影列表
4. 选择「全部平台」

**预期结果：**
- 选择特定平台后，列表只显示该平台的电影
- 选择「全部平台」后，显示所有电影
- 平台为空的电影，卡片上显示「未设置平台」

### 8. 统计更新

**测试步骤：**
1. 记录当前统计区的四个数值（总收藏、平均评分、今年观影、想看/已看/弃看）
2. 添加一部新电影，状态设为「已看」，评分 8.5
3. 观察统计区数值变化
4. 删除刚添加的电影
5. 观察统计区数值变化

**预期结果：**
- 添加电影后：
  - 总收藏数 +1
  - 已看数 +1
  - 平均评分根据新评分重新计算
  - 如果观影日期是今年，今年观影数 +1
- 删除电影后，所有数值回到添加前的状态

### 9. localStorage 持久化

**测试步骤：**
1. 添加几部电影
2. 记录当前电影列表状态
3. 刷新浏览器页面（F5）
4. 观察电影列表是否保持不变
5. 打开浏览器开发者工具 → Application → Local Storage
6. 查看是否有 `movie-watchlist-storage` 键
7. 点击「重置筛选」按钮（如果有筛选条件）
8. 刷新页面，验证筛选状态是否持久化

**预期结果：**
- 刷新页面后，电影列表保持不变
- localStorage 中存在 `movie-watchlist-storage` 键，包含电影数据
- 筛选状态也会持久化保存

## 表单校验验证

### 电影名称不能为空

**测试步骤：**
1. 打开新增电影表单
2. 不输入电影名称
3. 点击「保存电影」按钮

**预期结果：**
- 表单无法提交
- 输入框下方出现红色错误提示：「电影名称不能为空」
- 保存按钮可能处于禁用状态

### 评分限制在 0-10 之间

**测试步骤：**
1. 打开新增电影表单
2. 拖动评分滑块到两端
3. 观察评分值范围

**预期结果：**
- 评分最小值为 0，最大值为 10
- 步进为 0.5
- 无法设置超出范围的评分

### 已看电影建议填写观影日期

**测试步骤：**
1. 打开新增电影表单
2. 将状态切换为「已看」
3. 不填写观影日期

**预期结果：**
- 观影日期标签旁显示「(建议填写)」
- 输入框下方出现绿色提示：「标记为已看的电影建议填写观影日期」

## 移动端 UI 验证

### 验证方法
使用浏览器开发者工具的设备模拟功能（如 Chrome DevTools 的 Device Toolbar），或在真实移动设备上访问。

### 测试设备尺寸
- 手机竖屏：375 x 667（iPhone SE）
- 手机竖屏：390 x 844（iPhone 12/13/14）
- 平板竖屏：768 x 1024（iPad）

### 验证要点

1. **头部区域**
   - Logo 和标题不溢出
   - 文字大小适中，清晰可读

2. **统计区域**
   - 四格统计卡片在手机上为 2x2 布局
   - 数字和文字不溢出卡片
   - 图标大小适中

3. **新增电影表单**
   - 表单控件宽度适配屏幕
   - 输入框和按钮可正常点击（手指触控区域足够大）
   - 状态按钮在窄屏下不拥挤

4. **筛选区域**
   - 状态标签可换行显示
   - 下拉选择框在窄屏下堆叠
   - 文字清晰可读

5. **电影卡片**
   - 卡片宽度适配屏幕
   - 标题不超出卡片范围
   - 编辑和删除按钮可点击
   - 删除确认对话框完整显示

6. **整体布局**
   - 无横向滚动条
   - 各区域间距合理
   - 内容不拥挤
